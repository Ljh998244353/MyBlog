这个仓库不是一个通用模板，而是我自己的静态博客工作区。它的目标很直接：用尽量低的维护成本，把写作、预览、发布和后续微调串成一条稳定流程。

## 仓库里最重要的几个位置

- `src/content/blog/`：文章内容。每篇文章一个 `.mdx` 文件，文件名会直接成为文章 URL 的一部分。
- `src/pages/`：站点页面和静态数据接口，例如文章列表、搜索索引、RSS 和标签页。
- `src/components/`：布局、文章卡片、目录、搜索面板和 MDX 组件。
- `src/config/site.ts`：站点标题、描述、导航和站点 URL。
- `.github/workflows/deploy.yml`：GitHub Pages 自动部署流程。

如果只想“发一篇文章”，大多数时候只会碰到 `src/content/blog/`。如果想改导航、站点名、RSS 或 canonical URL，则要看 `src/config/site.ts`。

## 平时怎么发文章

我的日常流程已经压缩到两步：

```bash
npm run publish
git push
```

其中 `npm run publish` 会按顺序执行：

1. `astro check`
2. `astro build`

如果检查和构建都通过，再 `git push` 到 `main`。推送之后，GitHub Actions 会自动执行部署工作流，把静态产物发布到 GitHub Pages。

## 新文章应该怎么写

新文章放在 `src/content/blog/`，最少包含下面这些 frontmatter：

```md
---
title: 标题
description: 一句话摘要
pubDate: 2026-04-16
tags:
  - 示例
draft: false
---
```

如果只是写正文，Typora 和 VS Code 都能胜任；如果文章里用了 MDX 组件，例如提示框、实验说明或自定义片段，最终效果要以本地 `npm run dev` 或 `npm run publish` 的结果为准。

## 常见微调入口

### 1. 改站点基础信息

修改 `src/config/site.ts`：

- `title`：站点标题
- `description`：首页和 SEO 描述
- `siteUrl`：正式站点地址
- `navigation`：顶部导航

### 2. 改页面和样式

- 页面结构在 `src/pages/`
- 公共布局和卡片在 `src/components/`
- 全局样式在 `src/styles/global.css`

如果只是想换颜色、字重、间距或文章列表样式，优先改样式文件，不要在多个组件里重复写局部覆盖。

### 3. 改部署方式

当前默认部署到 GitHub Pages，工作流已经写在 `.github/workflows/deploy.yml`。如果将来改到别的平台，核心要调整的通常只有：

- 构建命令
- 产物目录
- 站点 URL 和 base path
