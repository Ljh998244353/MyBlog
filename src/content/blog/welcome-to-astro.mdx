Astro 很适合从零开始做博客，因为它把重点放回到内容本身，而不是把整个站点都变成前端应用。

## 为什么选 Astro

我需要的不是一个复杂的应用壳，而是一套稳定的内容发布流程：

- 写文章时可以专注于 Markdown 或 MDX
- 发布时输出纯静态文件
- 页面默认很轻，不必为了几段文字加载一堆客户端 JavaScript

## 首版目标

首版只做会被长期使用的能力：文章列表、详情页、标签页、RSS、SEO 基础和搜索。

```ts
export async function getPublishedPosts() {
  const posts = await getCollection('blog', ({ data }) => !data.draft);
  return posts.sort((a, b) => b.data.pubDate.getTime() - a.data.pubDate.getTime());
}
```

上面这段逻辑体现了一个核心原则：所有展示都围绕统一的内容源来生成，而不是在页面里各写一套筛选规则。

## 接下来会继续补什么

后续如果要继续增强，我更可能补的是写作体验，例如摘要模板、专题页和更细的内容归档，而不是过早引入评论系统或后台服务。
