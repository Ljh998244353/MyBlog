这篇文章记录当前博客切换到 Momo 模板后的写作与管理方式，方便以后新增文章时保持统一结构。

## 文章放置位置

文章正文放在 `src/data/blog/` 目录下，可以继续按主题创建子目录。例如：

```text
src/data/blog/Blog/momo-template-guide.md
src/data/blog/Cyberspace-security/ch1-Overview.md
```

文件名和所在目录会参与生成文章 URL。建议目录和文件名使用英文或拼音，文章标题继续写在 frontmatter 的 `title` 字段里。

## Frontmatter 示例

每篇文章顶部都需要写 frontmatter：

```yaml
---
title: 文章标题
description: 文章摘要，会显示在首页文章卡片中
pubDatetime: 2026-04-26T17:23:59+08:00
modDatetime:
tags:
  - 标签一
  - 标签二
draft: false
---
```

常用字段说明：

- `title`：文章标题。
- `description`：文章摘要。
- `pubDatetime`：发布时间，可以先留空为 `pubDatetime:`，本地检查或构建前会自动补当前时间。
- `modDatetime`：修改时间，可以留空。
- `tags`：文章标签，支持多个。
- `draft`：是否为草稿，发布文章时设为 `false`。

## 多标签归档

现在归档页按 `tags` 数组生成标签筛选项。一篇文章可以同时出现在多个标签下：

```yaml
tags:
  - SWJTU
  - 网络空间安全
  - 课程笔记
```

归档页右侧标签栏可以同时选择多个标签，只要文章包含任意一个已选标签，就会显示在结果中。文章卡片和文章详情页也会展示全部标签，并链接到对应的归档筛选结果。

## 精选标签

如果希望一篇文章进入首页的“推荐阅读”，在 `tags` 中加入 `精选` 即可：

```yaml
tags:
  - 博客
  - 使用说明
  - 精选
```

也可以继续使用旧字段：

```yaml
featured: true
```

当前首页会同时识别 `featured: true` 和 `精选` 标签。首页“推荐阅读”最多显示 3 篇精选文章，并保留人工推荐能力：先按 `pinTop` 倒序，再按发布时间倒序。

如果希望某篇精选文章在首页靠前，可以写：

```yaml
tags:
  - 博客
  - 精选
pinTop: 10
```

`pinTop` 只影响首页“推荐阅读”的排序，不改变文章文件的物理存储目录。

## 首页与列表页

当前首页不分页，只保留两个入口区：

- “推荐阅读”：最多 3 篇精选文章。
- “最近更新”：最多 3 篇非精选文章。

“推荐阅读”下方会跳转到 `/featured/`，这个页面显示所有精选文章，每页 6 篇，按发布时间倒序排列。

“最近更新”下方会跳转到 `/posts/`，这个页面显示全部文章，包括精选文章，每页 6 篇，按发布时间倒序排列。

归档页 `/archives/` 继续用于标签筛选和多标签归档；`/featured/` 和 `/posts/` 只负责按时间展示文章列表。

## 当前模板约定

当前站点已移除评论功能和友链页面；深色模式沿用当前博客配色方案。后续新增文章只需要维护文章内容和 frontmatter，不需要配置评论或友链相关字段。
